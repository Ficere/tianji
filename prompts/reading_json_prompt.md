# 步骤十：输出 reading.json

你现在已完成所有命理计算（chart.json 已就绪）。接下来输出符合 `schemas/reading_v8.schema.json` 规范的 JSON 对象。

## 输出格式规则

1. 输出纯 JSON，**不得**包含任何 Markdown 代码块标记（不写 ```json 或 ```）
2. JSON 对象输出完毕后，另起一行输出 `<!-- END_READING_JSON -->` 作为结束标记
3. JSON 内部**不得**写注释；不输出 schema 本身；不输出字段说明

## 数值来源规则

- 所有**数值型字段**（五行 scores、骨重克数、四柱干支、置信度）必须与 `chart.json` 保持一致，不得重新计算
- 所有**文字型字段**（analysis、interpretation、sketch 等）由你负责撰写

## 字段填写规则

| 字段 | 规则 |
|------|------|
| `meta.scene` | 仅1人填 `"personal"`，合盘填 `"synastry"` |
| `meta.version` | 固定填 `"8.0"` |
| `meta.title` | 个人：`"XX的命理报告"`；合盘：`"XX × XX 合盘"` |
| `name` | 仅提供真实姓名时填写，否则省略此 key（不填 null）|
| `birth_city` / `rising_sign` | 无城市信息时省略此 key |
| `name_analysis` | 提供真实姓名时填完整对象，否则填 `null` |
| `synastry` | 个人命盘时填 `null`；合盘时填完整 SynastryReading 对象 |
| `life_palace_empty` | 命宫空宫时填 `true` 并填写 `empty_palace_note` |

## 置信度评分标准

| 情况 | 分值 |
|------|------|
| 出生时间精确到分钟 + 提供出生城市 | 5.0 |
| 精确到时辰 + 提供出生城市 | 4.5 |
| 精确到时辰，无城市 | 4.0 |
| 仅知年月日，时辰未知 | 3.0 |
| 年月已知，日期不确定 | 2.0 |
| 仅知出生年份 | 1.0 |

- `western_astro` 置信度：有城市+精确时间则同八字；无城市则降1星；无时辰则降2星
- `personality_sketch` 置信度：取所有维度中最低置信度值
- **禁止**所有维度都给 5 星

## 合盘 composite_scores 口径

> 五行平衡/20 + 五行俱全/5 + 生肖/20 + 星座/15 + 日主/20 + 称骨/15 + 姓名/5 = **100**

- 星座相位分区间：0/60/120°=13-15分；30°=10-12；180°=7-9；90°=4-6；150°=1-3
- `total` 必须等于各维度 score 之和，不得有差异

## 文字风格规范

- 语气：像有主见的朋友在分析，不像老师在讲课
- 每个特征必须同时呈现优势与风险，不得只写正面
- 禁止使用「XX方面很有潜力」「未来可期」等空洞表述
- `personality_sketch` 三层必须融合八字/紫微/星座三系统，不得只复述单一体系
- 文字长度：各 `_analysis` / `_interpretation` 字段 2-6 句，不超过 200 字

## 输出流程

1. 输出完整 JSON 对象（一个连续的 JSON，不分段）
2. 换行输出 `<!-- END_READING_JSON -->`
3. 再输出 1-2 句面向用户的摘要话语（非 JSON，普通中文）
